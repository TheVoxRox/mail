# Translation whitelist
#
# Files in this list may keep Czech text in comments, logs, or string literals
# because the Czech content is load-bearing (e.g. SMTP-bound output, fixed
# user-visible legal copy, etc.). Every entry MUST be justified.
#
# Format: one path per line, relative to repo root. Comments start with '#'.
# Blank lines ignored.
#
# Lint script: frontend/scripts/check-translation-whitelist.mjs --target=backend
#
# Policy (from todo.md "Migrace komentaru a logu do anglictiny"):
#  - Programmer text (comments, log messages, internal exception messages)
#    in backend/src/main/java MUST be English.
#  - User-facing Czech that ships to recipients (SMTP body, legal text)
#    stays Czech and lives on this whitelist.
#  - i18n message bundles (messages_cs.properties etc.) are the SOURCE of
#    translations and are not lint targets.
#
# When adding an entry: include reason on the same line after '--' or on a
# preceding comment block. When the Czech content is removed, delete the entry.

# --- Permanent whitelist ---

backend/src/main/java/org/voxrox/mailbackend/feature/mail/service/MailDraftService.java -- SMTP-bound Czech labels in reply/forward quoted body ("Puvodni zprava", "Preposlana zprava", "Od:", "Datum:", "Predmet:") shipped to Czech recipients. Comments and log messages in this file should still be translated; only the string literals stay.
backend/src/main/java/org/voxrox/mailbackend/feature/mail/dto/FolderRole.java -- Czech substring "koš" in fromNameFallback(...) is a load-bearing match against real Czech IMAP folder names returned by Czech mail providers (e.g. Seznam.cz "Koš"). Removing it would mis-classify the trash folder. Other Czech roots in the same switch are diacritic-free (odeslan, smazan, rozeps, archiv) and slip past the lint.

Removing the diacritic would weaken the rationale documented for the name-based override.
backend/src/main/java/org/voxrox/mailbackend/feature/mail/service/MessageFetcher.java -- Single Javadoc example demonstrates RFC 2047 encoded-word decoding (`=?utf-8?q?Tren=C3=BDrk=C3=A1rna?=` → `Trenýrkárna`). The Czech glyph is the decoded form of the encoded-word in the same line and is load-bearing for the example.
