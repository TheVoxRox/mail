# Translation whitelist (frontend)
#
# Files in this list may keep Czech text in comments, strings, or fixtures
# because the Czech content is load-bearing (e.g. test assertions against a
# Czech UI locale, mock backend data simulating Czech-language responses,
# transliterated command palette aliases). Every entry MUST be justified.
#
# Format: one path per line, relative to repo root. Comments start with '#'.
# Blank lines ignored. Reasons go after '--' on the same line.
#
# Lint script: frontend/scripts/check-translation-whitelist.mjs
#
# Policy (from todo.md "Migrace komentaru a logu do anglictiny — FRONTEND"):
#  - Programmer text (comments, log messages, pre-i18n error strings) in
#    frontend/src/lib/ and frontend/src/routes/**/*.{ts,svelte} MUST be
#    English.
#  - User-facing translated UI text lives in lib/i18n/messages/{cs,en}.json
#    only; cs.json is the SOURCE of Czech translations, not a lint target.
#  - E2E tests assert against real Czech UI text under mail.locale=cs;
#    refactor to locale-agnostic selectors is a separate task.
#  - MSW test fixtures simulate backend ProblemDetail responses with the
#    Czech `detail` payload that the prod backend emits under
#    Accept-Language: cs; translating them would defeat the i18n test path.
#
# When adding an entry: include reason on the same line after '--'. When the
# Czech content is removed, delete the entry.
